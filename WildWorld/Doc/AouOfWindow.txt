In MonoGame, mouse clicks can sometimes "pass through" the game window to applications behind it, or continue to register when the mouse cursor is outside the game window. 
Here are the primary methods to prevent this behavior based on the community and official documentation:
1. Check IsActive (Best Method) 
The Game.IsActive property tracks whether your game window is currently focused. If the user clicks on another window, IsActive becomes false. You should check this in your Update method for both mouse and keyboard input to prevent accidental input handling when the game is in the background.

2. Validate Mouse Coordinates
If you want to allow the game to run in the background but only process clicks inside the window boundaries, compare the MouseState coordinates against your backbuffer size.

MouseState mouseState = Mouse.GetState();
if (mouseState.X >= 0 && mouseState.Y >= 0 && 
    mouseState.X < GraphicsDevice.Viewport.Width && 
    mouseState.Y < GraphicsDevice.Viewport.Height)

    // Mouse is inside the window
}
3. Handle Deactivated Event 
Use the Deactivated event to pause your game or disable input handling when the window loses focus.

4. Lock Cursor to Window (DesktopGL)
For desktop applications (specifically OpenGL), you can use Mouse.SetPosition to keep the mouse within bounds, though this can sometimes cause stuttering (jitter) if not implemented carefully. A better approach for FPS-style games is to hide the system cursor and use a virtual cursor that is clamped to the window boundary.
